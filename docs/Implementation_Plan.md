## Неделя 4: Аналитический слой (Mart)
- **Справочники:** Создан справочник категорий праздников `holiday_types.csv`.
- **Соединение:** Выполнен `left join` с валидацией `many_to_one`.
- **Витрина:** Сформирована ежемесячная витрина (Mart).
- **KPI:**
  1. Общее количество праздников в месяц.
  2. Количество государственных выходных (`is_work_free`).
  3. Скользящее среднее количества праздников (окно 3 месяца).

## Неделя 5: Инфраструктурный слой (Database & ETL)
- **Контейнеризация:**  Развернута база данных PostgreSQL в изолированном Docker-контейнере.
- **Библиотеки:** Настроен стек технологий для работы с БД: SQLAlchemy (ORM) и драйвер psycopg2-binary.
- **Автоматизация загрузки (ETL):**  Разработан Python-скрипт load.py, который автоматически находит последнюю версию витрины (Mart) и загружает её в базу данных.
- **Валидация (Data Quality):**
1. Написаны SQL-запросы для проверки целостности данных после загрузки.
2. Реализована автоматическая проверка количества строк (валидация 10 строк витрины).
3. Выполнена проверка на отсутствие пустых значений (NULL) и дубликатов по месяцам.

## Неделя 6: Автоматизация и идемпотентность
- **Оркестрация**: Реализован центральный управляющий скрипт `pipeline.py`, обеспечивающий последовательный запуск всех этапов ETL (Extract -> Transform -> Load).
- **Управление состоянием (Watermark)**:
    - Внедрено отслеживание состояния через `data/state/state.json`.
    - **Watermark**: Поле `last_year` фиксирует последний успешно обработанный год.
    - Состояние обновляется только после успешного завершения финального этапа загрузки (`load.py`).
- **Инкрементальность и Идемпотентность**:
    - **Режим Incremental**: Пайплайн проверяет текущий год против watermark и пропускает этап Extract, если данные уже успешно обработаны.
    - **Режим Full**: Реализован флаг `--mode full`, позволяющий игнорировать состояние и принудительно перезапустить весь цикл обработки данных.
    - **Business Key**: Уникальность записей в базе данных обеспечивается ключом `(year, holiday_name)`, а использование метода `replace` при загрузке исключает дублирование аналитических результатов.
- **Интерфейс командной строки (CLI)**:
    - Поддержана параметризация через библиотеку `argparse`.
    - Команда запуска: `python src/sem2_de/pipeline.py --config configs/variant_20.yml --mode incremental`.
- **Слои данных**: Артефакты разделены по физическим папкам: `data/raw/` (грязные данные), `data/normalized/` (очищенные CSV) и `data/mart/` (финальные витрины).
